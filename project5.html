📘 作品五：LangChain RAG 法規問答系統

📘 內容說明

本系統是以 LangChain + Streamlit 實作的 RAG（Retrieval-Augmented Generation）法規問答平台，使用者可上傳車輛法規 PDF 並用自然語言提問，由 AI 模型根據檢索段落生成回應，有效提升查詢效率。

🧠 程式設計說明

  • 環境與模型初始化：
    • 載入 OpenAI API 金鑰
    • 初始化 GPT-3.5 模型與向量嵌入模型（OpenAIEmbeddings）
  • PDF 上傳與處理：
    • 上傳法規 PDF（如 CarRegulations_True_or_False_Questions.pdf）
    • 使用 PyPDFLoader 分割為多個段落 chunks
  • 建立向量資料庫：
    • 使用 text-embedding-3-large 模型將文字轉為向量
    • 儲存在本地 Chroma 向量資料庫中
  • 問答與檢索：
    • 使用者輸入問題，Retriever 擷取最相關段落
    • GPT 模型整合 context 與問題後生成回答
  • 展示與下載：
    • 顯示 AI 回答與參考段落
    • 可下載回答（rag_result_*.txt）

✍️ 學習應用心得

  • 🔍 深化對 RAG 架構的理解： RAG 是結合文件檢索與生成式 AI 的利器，適合回答特定資料集的查詢任務。
  • 📚 強化資料導向的 AI 應用思維： 使用檢索限制模型回答內容，減少幻覺（hallucination）問題。
  • 🖥️ 學會模組整合協作： 從嵌入、查詢到生成與前端展示，完整實作 AI 工具鏈。
  • 🛠️ 擴充性高： 可延伸應用至：
    • 法律諮詢（勞基法、智慧財產權）
    • 醫療規範或指引查詢
    • 校內行政規章自助系統

這個專案不僅讓我學會實作 RAG 系統，也真正體驗了 AI 如何強化現有資料的查詢與應用方式，是我學習生成式 AI 應用中非常具代表性的經驗之一。
